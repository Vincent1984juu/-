

### 2026-06-01 钉钉输出格式问题深度排查
- **问题**：钉钉上所有回复（包括正常对话和技能回复）都变成"✅ 媒体已发送"，结构混乱
- **排查过程**：
  1. 先修改报销技能模板（SKILL.md），从空一行改为空两行 → 无效
  2. 发现根源在钉钉插件本身：`streamFromGateway` SSE 解析有问题，`cleanMarkdownForDingTalk` 虽然清理了 Markdown 但可能过度清理
  3. 日志分析发现：`Gateway 流完成，共 0 chunks, 0 字符` 反复出现，说明 SSE 流没有返回任何内容
  4. `embedded run timeout` 和 `lane wait exceeded` 表示 AI run 超时或会话队列阻塞
  5. 发现 `streamFromGateway` 在 `reader.read()` done=true 时丢弃了 buffer 中剩余数据
- **修改内容**：
  1. 新增 `cleanMarkdownForDingTalk` 函数，发送前自动清理 Markdown 语法
  2. 在 `buildMediaSystemPrompt` 中注入钉钉格式 system prompt（纯文本、数字列表、空行分隔）
  3. 修复 `streamFromGateway` 的 SSE 解析：处理 `reader.done` 时 buffer 中剩余数据
  4. 在 AI Card 流式、AI Card 完成、普通消息、主动发送 4 个出口都应用清理
- **状态**：修改已保存到 `/root/.openclaw/extensions/dingtalk-connector/plugin.ts`，gateway 已重启加载
- **待验证**：需用户在钉钉上测试，确认格式是否正常
