# 报销单整理技能

## 触发与指引

当用户说"帮我整理报销单"、"报销单"、"整理报销"、"整理凭证"、"报销"、"帮我报销"、"开启新的报销"、"开启报销"等时：

**第一步：发送指引消息**

回复以下内容（钉钉格式：段落间空两行、数字列表、短句）：

```
好的，开启新报销！


请提供报销凭证截图。


一次可以发多张。


【建议分批发】


1. 先发同一类型


2. 比如先差旅、再原材料


3. 这样准确性更高


【截图里要有】


1. 金额


2. 日期


3. 用途信息


【信息不足时】


文字备注一下。


比如：图2是停车费。


【你只需做】


1. 发截图


2. 补充我识别不全的信息


3. 指出我整理有误或遗漏的地方


发图过来吧！
```

**第二步：等待用户提供截图**

用户可能：
- 直接发图片（之前已完成指引，继续发图）
- 发文字+图片混合
- 分多轮发图

## 处理流程

### 1. OCR识别

对每张截图：
- 用 `image` 工具分析图片内容
- 或用 `python3 /root/.openclaw/workspace/ocr_tool.py <图片路径>` 提取文字
- 优先用 `image` 工具，如果失败再用OCR脚本

**提取字段：**
- 日期（截图上的日期，各种格式）
- 金额（数字，¥/￥/元 符号附近）
- 商家/用途关键词（用于分类）
- 交通类：起点、终点、里程（km）、如果是导航截图无金额则记录里程数
- 其他可见信息

### 2. 智能分类

**一级类型**（7个）：
- 原材料
- 差旅
- 工资
- 租金水电物管
- 福利费
- 办公
- 运输费

**二级类型判断规则（按优先级）：**

**分类逻辑：**
1. 先根据关键词匹配二级类型
2. 二级确定后，一级类型自动确定
3. 如果一张截图无法判断，记录为"待确认"，并在Word中标注
4. 多张截图同类合并，但最终每条单独一行（方便核对）

**交通费特殊处理：**
- 如果是导航/地图截图，没有显示金额 → 按里程×1元/km计算费用
- 如果显示了金额，用截图上的金额
- 必须记录起点和终点（从截图中提取）

### 3. 分类与确认流程

**关键规则：先确认，后生成。**

完成OCR识别和分类后，**先检查是否存在以下待确认项**：
- 分类不确定 → 标记"待确认"，给出判断依据
- 金额缺失 → 标记"金额待补"
- 日期缺失 → 标记"日期待补"
- 用途不明（如给个人转账）→ 标记"用途待确认"

**流程分叉：**

**情况A：存在待确认项**

停止生成，先输出确认清单（钉钉格式）：

```
已识别X张凭证，发现以下待确认项：


【待确认清单】


1. 图X：金额缺失


金额待补，请补充。


2. 图X：分类待确认


判断依据：截图显示为个人转账，无法识别用途。


建议归为：差旅-交通费


请确认是否正确。


3. 图X：日期待补


未显示日期，请补充实际日期。


确认或补充后，我再生成报销单！
```

等待用户回复确认/补充，**再执行步骤4生成Word**。

**情况B：全部确定**

直接执行步骤4生成Word。

### 4. 生成Word报销单

用 `scripts/generate_reimbursement_word.py` 生成，内部逻辑：
1. 先按日期从远到近排序所有明细
2. 按二级分类分组聚合
3. 每份Word中，同二级分类的明细按日期排列，末尾插入小计行

**文档格式**：

- **标题**：如"差旅报销单"、"原材料报销单"
- **表格列**：序号、日期、一级类型、二级类型、金额(元)、合计(元)、凭据截图、费用说明、起点、终点
- **二级小计**：同个二级分类的所有明细行下方插入灰色小计行，标注"XX小计"及金额
- **总计行**：每份Word底部蓝色总计行，金额=各二级小计之和
- **图片嵌入**：凭据截图按宽高比例缩放到2.5cm×1.8cm以内嵌入单元格
- **交通费特殊处理**：仅差旅类填写起点终点，其余留空

**样式要求：**
- 表头蓝色背景+白色字体+加粗
- 金额列右对齐，保留2位小数
- 小计行灰色背景+粗体，总计行蓝色背景+粗体
- 列宽自适应
- 日期统一格式：YYYY-MM-DD

**输出规则**：
- 不同一级分类，各生成**独立Word文档**（如差旅.docx、原材料.docx、办公.docx）
- 同个一级分类内的所有二级分类明细合并到同一文档
- 文档命名：`报销单_<一级类型>_<日期>.docx`
- 日期用YYYY-MM-DD格式

### 5. 输出与确认

生成Word后发送文件，并发送汇总文字（钉钉格式）：

```
报销单整理完成！


共X条记录。


【差旅报销单】


X条，合计XX元


1. 车费：X条，小计XX元


2. 餐费：X条，小计XX元


3. 停车费：X条，小计XX元


...


【福利费报销单】


X条，合计XX元


1. 工作餐：X条


【办公报销单】


X条，合计XX元


1. 招聘费：X条


【原材料报销单】


X条，合计XX元


1. 其他材料：X条


【总计】


XX元


如有分类不准确，请告诉我，我重新调整。
```

**钉钉消息格式要求（非常重要）：**

钉钉AI Card对换行支持很弱，消息必须以**纯文本+空行**方式输出：

- 每句话、每段之间**必须空一行**（两个换行符）
- 列表项之间**也必须空一行**
- 用**数字+句点**格式，不要用减号
- 关键信息（如金额）用【】括起来
- **禁止使用markdown表格**（钉钉移动端不渲染）
- **禁止减号列表**（显示效果差）
- **禁止一大段连续文字**，必须拆成短句

核心原则：**宁可多空行，不要少空行。** 空行在钉钉上不会显得太稀疏，但少了就会糊成一团。

**正确的钉钉格式示例：**

```
好的，开启新报销！


请提供报销凭证截图。


一次可以发多张。


【建议分批发】


1. 先发同一类型


2. 比如先差旅、再原材料


3. 这样准确性更高


【截图里要有】


1. 金额


2. 日期


3. 用途信息


【信息不足时】


文字备注一下。


比如：图2是停车费。


发图过来吧！
```

**错误的格式（不要这样输出）：**

- 不要写成"好的，开启新报销。请提供报销凭证截图。建议一次发一个类型：1.先发差旅费 2.等处理完再发原料费..."（没有换行，连成一段）
- 不要用markdown表格

**特别说明：**
- **费用说明列默认留空**，只有用户明确备注说明时才填写
- **所有明细按日期从远到近排序**，同个二级分类内的明细也保持日期顺序
- 如果分类不确定，标注"待确认"并给出建议
- 交通费导航截图无金额时，说明按1元/km计算

## 边界情况

- 用户发了无文字的截图（如纯logo）→ 标记为"无法识别"，列入待确认清单，先问用户
- 金额缺失 → 标记为"金额待补"，不生成，先问用户补充
- 日期缺失 → 标记为"日期待补"，不生成，先问用户补充
- 分类不确定 → 标记为"待确认"，给出判断依据，不生成，先问用户确认
- 用户要求修改分类 → 修正数据后重新生成Word
- 用户补充信息 → 更新数据后重新生成Word

## 流程回顾

1. 接收截图

2. OCR识别

3. 智能分类

4. 检查待确认项

有待确认 → 输出确认清单 → 等用户回复 → 确认/补充后 → 生成Word

全部确定 → 直接生成Word

5. 发送汇总消息 + Word文件

## 工具使用

- **OCR**：`python3 /root/.openclaw/workspace/ocr_tool.py <图片路径> chi_sim` 或 `image` 工具
- **Word生成**：`python3 /root/.openclaw/workspace/skills/reimbursement-organizer/scripts/generate_reimbursement_word.py <json数据> <图片目录> <输出目录>`
  - 按一级分类自动分组，每组输出独立Word文档
  - 同个文档内按二级分类聚合，每类末尾插入小计行
  - 凭据截图自动缩放到单元格内

## 文件输出

- **只输出Word文档**，每个一级分类各一份
- 类型用用户当前批次的主要一级类型命名，如"差旅"、"原材料"等
- 日期用YYYY-MM-DD格式
- 示例：`/root/.openclaw/workspace/报销单_差旅_20260529.docx`
