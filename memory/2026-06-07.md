## 营业额分析报告格式优化（钉钉AI表格同步）

### 问题
用户反馈钉钉AI表格中报告原文格式难看：
1. 中文序号被截断（"一、"只剩"、"）
2. 内容挤在一起，换行被压缩
3. 数字列表渲染成 ● 圆点，折行缩进混乱
4. 加粗标记 `**` 行内不生效，显示为文本

### 根因
钉钉AI表格的 `richText` 字段类型对 markdown 渲染行为特殊：
- 字段类型必须是 `markdown`（之前误传 `text` 导致内容为空）
- 只支持**整行加粗**，不支持行内加粗（`**xxx**内容` 会显示 `**`）
- 数字列表自动转成 ● 圆点，缩进不可控
- 中文序号（一、二、三）可能被截断
- 连续换行会被压缩

### 解决方案
1. **字段类型修正**：`dingtalk_ai_table.py` 中从 `text` 改回 `markdown`
2. **中文序号 → 【章节名】**：如 `**【基础信息】**` `**【数据总览】**`
3. **数字列表 → 【动作编号】**：如 `**【动作1】**` 单独一行，内容另起
4. **【根源】单独一行**：`**【根源】**` 换行后写内容，避免行内加粗失效
5. **每个数据项独占一行**：目标/实际/达成率/环比/同比各一行
6. **段落之间空两行**：确保视觉区块分明

### 修改文件
1. `dingtalk_ai_table.py`：
   - 新增 `format_report_for_dingtalk()` 函数，自动转换格式
   - 字段类型 `markdown`（修正之前误传的 `text`）
   - 保留加粗标记，由脚本和模板共同控制格式

2. `skills/monthly-revenue-analysis/SKILL.md`：
   - 报告模板完全重写，适配钉钉渲染规则
   - 明确禁止：markdown表格、行内加粗、中文序号、数字列表、减号列表
   - 明确保留：整行加粗 `**【标签】**` 单独一行

### 经验
钉钉AI表格的markdown支持非常有限，不能按标准markdown思维写。关键规则：
- 整行加粗 ✅  行内加粗 ❌
- 纯文本分段 ✅  列表符号 ❌
- 【】大标题 ✅  中文序号 ❌
- 每行一个信息 ✅  多信息挤一行 ❌

## 当日关键教训

1. **用户说"可以"不等于"全部确认"**——用户只确认了3个动作（动作1、4、5），我却在报告里加了4个动作（多写了新会员激活）。修正后重新生成报告。以后要逐个确认用户同意的动作，不多写。

2. **截图识别要区分凭证和产品图**——用户发的蛋糕图不是报销凭证，需要及时提醒用户。

3. **营业额分析对话要分批跟进**——多个用户同时在做分析，每次发数据后要明确告诉用户下一步需要什么，避免对话中断后丢失上下文。

4. **功能菜单格式**——用户经过3轮测试才定稿（纯文本→【】标题→加粗章节+数字序号），以后先问用户偏好再生成，减少来回修改。
