# 培训教材：从"拍脑袋"到"神算子"——包装产品（成品）精准订货管理

> 来源：用户上传《如何订准包装包.docx》
> 适用：门店包装产品（预包装吐司、常温饼干、礼盒等）订货管理
> 入库时间：2026-06-17

---

## 一、痛点引入：为什么我们总是订不好货？

在我们这样拥有50多家门店的连锁体系中，任何一个店长在订购包装类成品时的"拍脑袋"，都会在整体大仓数据上被成倍放大。常见陷阱：

1. **凭感觉订货（拍脑袋）：** "我觉得明天周末人多，随便多订几箱吧。"
2. **复制粘贴法（抄作业）：** "上周订了多少，这周就点个'再来一单'。"
3. **只看系统不看实物（盲人摸象）：** 账面库存是5，实际仓库里还有一箱坏的没报废。
4. **不敢订货（因噎废食）：** 上次订多了报废多，这次干脆少订点，结果陈列面空空荡荡。

**核心认知：** 订多了叫**资金积压和损耗**，订少了叫**眼睁睁看着利润溜走**。精准订货是店长向利润要结果的必备核心技能。

## 二、订货的终极公式：精准填补"时间差"

在"当天订货，隔天到货（T+2）"模式下，所有包装成品订货套用以下公式：

> **🎯 周五建议订货量 = 周五日均销量 - 【（周三实际库存 + 周四来货量） - 周四日均销量】**

**两步走逻辑：**

**第一步（算括号里的）：推算周五早上的"开局库存"**
- 周三晚上的实地库存 + 周四早上送来的货 - 周四一天卖掉的货
- 剩下的就是**周五早上开店时手里还剩多少货**

**第二步（算缺口）：计算最终订货量**
- 周五一天打算卖的量 - 周五早上手里剩的量 = 要向大仓订的量

### 店长必抓的4个核心数据

| 数据 | 含义 | 获取方式 |
|------|------|---------|
| **目标日预估销量** | 打算卖多少 | 调取POS系统，过去2-3个同类型日子的平均销量 |
| **当下真实库存** | 实际有多少 | **必须实地手工盘点**，不能只看系统账面 |
| **途中的到货量** | 明天会来多少 | 查阅昨天下的订单 |
| **过渡日预估消耗** | 明天会卖多少 | 参考过去几个过渡日的平均销量 |

## 三、影响订货的5大关键变量（实战微调）

公式是死的，营运是活的。优秀店长必须把以下变量加入微调：

1. **天气与气温变量**
   - 恶劣天气需减量
   - 突然升温→冷柜饮品暴增
   - 突然降温→常温烘焙品需求上升

2. **营销与活动变量**
   - 总部买赠/换购活动？社群团购？
   - 必须根据转化率成倍增加备货

3. **保质期与周转率（包装产品的命门）**
   - 短保产品（7-15天）："少量多次"，严控库存
   - 长保产品：结合配送频次按周囤货

4. **节假日与商圈特殊节点**
   - 学校开学、社区活动、法定节假日
   - 根据往年同期数据拉高预估

5. **大仓起订量与包装规格**
   - 计算出缺口12个，但一箱15个
   - 结合保质期决定：向上取整凑一箱，还是向下抹零硬抗

## 四、包装产品标准订货4步法（SOP）

| 步骤 | 核心动作 | 关键注意事项 |
|------|---------|-------------|
| **1. 盘点实库** | 走到现场，眼见为实 | 查数量同时**必须查效期**，临期品挑出做促销，不计入可用库存 |
| **2. 分析数据** | 调取报表，提取同周期数据 | 提取目标日（T+2）与过渡日（T+1）的历史日均销量参考值 |
| **3. 叠加变量** | 加入大脑思考，灵活微调 | 结合天气预报、门店活动、商圈动态修正预估销量 |
| **4. 审核下单** | 二次核对，确认提交 | 店长对"金额大、占地大、保质期短"的高风险物资最终复核 |

## 五、高阶实战演练：当公式遇到"突发变量"

### 【背景案例】

今天是**周三晚上**，店长准备订购爆款"预包装红豆吐司"（保质期3天，周五早上到货）。基础数据：

- 周五历史均销：40袋
- 周四历史均销：30袋
- 途中来货（周四早）：25袋

**⚠️ 三个极具破坏力的变量：**

**变量A（临期陷阱）：**
- 系统库存20袋，但**5袋明天（周四）到期**
- 必须挑出做试吃或买一送一，不能留到周五卖

**变量B（天气预警）：**
- 周五大暴雨，常规客流锐减
- 销量打7折：40袋 × 0.7 = 28袋

**变量C（促销活动）：**
- 周五启动"买现磨美式+29元换购红豆吐司"
- 根据经验，额外带动**15袋**销量

### 【硬核推演过程】

**Step 1：修正基础数据**

- **周三真实可用库存：** 20袋 - 5袋（临期）= **15袋**
- **周五最终预估销量：** 28袋（大雨打折）+ 15袋（活动增量）= **43袋**

**Step 2：套入终极公式**

```
周五建议订货量 = 43 - 【 (15 + 25) - 30 】
```

1. **算开局库存：** (15 + 25) - 30 = **10袋**
   （周五早上开门，货架上还剩10袋可用）

2. **算最终缺口：** 43 - 10 = **33袋**

### 【如果不按这套方法】

如果"拍脑袋"直接用基础数据：

```
40 - 【(20 + 25) - 30】= 25袋
```

**结果灾难：**
- 促销爆发缺货8袋被投诉
- 货架上摆着5袋过期吐司面临食安通报

**结论：** 应当输入 **33**。做营运，**手要勤（查临期），脑要清（算变量）**，对数据的敏锐度就是店长的价值所在。

## 六、适用范围

以上方法不仅适用于**包装包**，同样适用于：
- 面团
- 半成品
- 其他T+2模式订货的成品

---

## 关键行动清单

1. **每天打烊前**：执行4步SOP（盘点→分析→叠加变量→审核下单）
2. **订货前必做**：实地盘点 + 查效期，临期品挑出做促销
3. **每周复盘**：对比预估 vs 实际销量，校准自己的预估偏差
4. **建立数据习惯**：记录天气/活动对销量的影响，形成门店自己的微调系数
